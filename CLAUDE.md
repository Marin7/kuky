# Kuky вЂ” EspaГ±ol con Paula

Personal website for Paula, a Spanish teacher targeting Romanian students.
A full-stack application with a React SSR frontend and a Java Spring Boot REST API.

## Tech stack

### Front-end

| Layer | Technology |
|-------|-----------|
| Framework | React 19 + TanStack Start 1.x (SSR) + TanStack Router (file-based) |
| Styling | TailwindCSS 4.x + Shadcn UI (Radix UI primitives) |
| Build | Vite 7 via `@lovable.dev/vite-tanstack-config` |
| Package manager | npm (bun preferred вЂ” install from https://bun.sh if available) |
| Language | TypeScript 5.x strict |
| Runtime | **Node.js 22+ or 24+** required (Vite 7 minimum; Node 20.17 is too old) |

### Back-end

| Layer | Technology |
|-------|-----------|
| Language | Java 26 |
| Framework | Spring Boot 4.0.x + Spring Security + Spring Data JPA |
| Database | PostgreSQL 18 (local: pgAdmin; production: managed PostgreSQL) |
| Migrations | Flyway (versioned SQL under `db/migration/`) |
| Auth | JWT in HTTP-only SameSite=Lax cookies, 7-day rolling session |
| Password hashing | BCrypt cost factor 12 |
| Rate limiting | Custom sliding-window counter (10 attempts / 15 min per IP) |
| Email | JavaMailSender + Mailpit (local SMTP capture) |
| Build | Gradle |


## Local dev setup

1. **PostgreSQL 18** вЂ” running via pgAdmin. Create the database and user once:
   ```sql
   CREATE DATABASE kuky_dev;
   CREATE USER kuky WITH PASSWORD 'kuky';
   GRANT ALL PRIVILEGES ON DATABASE kuky_dev TO kuky;
   GRANT CREATE ON SCHEMA public TO kuky;
   ALTER USER kuky WITH SUPERUSER;   -- simplest for local dev
   ```
2. **Mailpit** вЂ” SMTP capture for email testing. Run `mailpit` (port 1025 SMTP, http://localhost:8025 web UI).
3. **Backend** вЂ” `./gradlew bootRun --args='--spring.profiles.active=local'` вЂ” Flyway auto-runs migrations on first start.
4. **Frontend** вЂ” `npm run dev` from `front-end/`.

## Key implementation notes

### Front-end
- **Image imports**: Use standard Vite imports (`import url from "@/assets/foo.jpg"`),
  NOT Lovable-specific `.asset.json` pattern вЂ” that only works inside Lovable's cloud.
- **Routing**: File-based via TanStack Router. `routeTree.gen.ts` is auto-generated;
  never edit it manually. Add a new page by creating `src/routes/<name>.tsx`.
- **Active nav links**: Use `activeProps` + `activeOptions={{ exact: true }}` on `<Link>`
  for the home route (`/`); other routes don't need `exact`.
- **Mobile nav**: `SiteHeader.tsx` shows a hamburger button on `md:hidden` that opens a
  Shadcn `Sheet` with the same five nav links.
- **SSR**: TanStack Start renders all pages server-side. Client-only code must live in
  event handlers or `useEffect`.
- **Fonts**: Fraunces (display/headings via `font-display` class) and Inter (body) loaded
  from Google Fonts in `__root.tsx`.
- **Auth (client-side)**: `cuenta.tsx` manages auth state via `useEffect` + `getMe()`.
  All API calls go through `src/lib/auth.ts` with `credentials: 'include'` for cookie handling.

### Back-end
- **Auth cookie**: Named `auth-token`, `HttpOnly`, `SameSite=Lax`, `maxAge=604800` (7 days).
  Rolling refresh: every authenticated request resets the MaxAge (handled in `JwtCookieAuthenticationFilter`).
- **Email normalisation**: Stored and compared in lowercase (FR-018).
- **Password reset tokens**: UUID v4, 1-hour expiry, single-use. Previous unused tokens
  invalidated on new request and on consumption.
- **Rate limiting**: `LoginRateLimiter` вЂ” sliding window, 10 attempts per 15 min per IP,
  in-memory `ConcurrentHashMap<String, Deque<Long>>`. No external dependency.
- **Error codes**: All API errors return `{"error":"ERROR_CODE","message":"..."}`.
  Codes defined in `contracts/api.md` and mapped in `GlobalExceptionHandler`.
- **Local mail**: When Mailpit is not running, `EmailService` logs the reset URL to the
  console (`WARN EmailService вЂ” password reset URL for ...`).

## Production environment variables

The back-end reads these at startup (set in your deployment environment):

| Variable | Description |
|----------|-------------|
| `DB_URL` | JDBC URL, e.g. `jdbc:postgresql://host:5432/kuky` |
| `DB_USERNAME` | Database username |
| `DB_PASSWORD` | Database password |
| `APP_JWT_SECRET` | HS256 signing key, at least 32 characters |
| `CORS_ALLOWED_ORIGIN` | Frontend origin, e.g. `https://kuky.es` |
| `FRONTEND_BASE_URL` | Used in password reset emails, e.g. `https://kuky.es` |
| `MAIL_HOST` | SMTP host (defaults to `localhost`) |
| `MAIL_PORT` | SMTP port (defaults to `1025`) |
| `MAIL_FROM` | From address (defaults to `noreply@kuky.es`) |
